# 模块 README 模板（A-023）

每个后端领域模块在 `modules/<name>/README.md` 复制本模板并填写，使未参与实现者
也能据此理解模块边界。模板包含五个必填小节：职责、入口、状态、依赖、禁区。

> 复制本文件为 `modules/<name>/README.md`，替换 `{{...}}` 占位符为该模块的实际内容；
> 删除模板说明性文字，保留五个小节标题。

---

# {{模块名}}（{{module-name}}）

> 一句话职责说明。

## 职责

- 本模块负责的领域概念与用例（列出主要聚合、命令与查询）；
- 不承担的职责（指向其他模块）。

## 入口

- API 路由：`{{prefix}}`，主要端点与用例；
- 应用用例：`{{use case 列表}}`；
- 异步任务类型（若有）：`{{task_type}}`。

## 状态

- 涉及的状态机（引用 `shared/states.py` 中的枚举）；
- 关键状态转换与终态。

## 依赖

- 上游模块（被本模块依赖）；
- 端口（application 声明、infrastructure 实现）；
- 共享内核（`shared/` 中使用的值对象与契约）。

## 禁区

- 不得跨模块直接读写 ORM Model；
- domain 层不得导入 Web/ORM/队列/供应商 SDK（由 A-006 架构检查守护）；
- 不引入用户/组织/角色/Token/RBAC/租户字段或兼容分支（SPEC.md 第 4 节、ADR-014）。
