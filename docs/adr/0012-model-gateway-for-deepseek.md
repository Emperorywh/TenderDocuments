# ADR-012 DeepSeek 通过 ModelGateway 适配

- 状态：Accepted
- 日期：2026-07-22

## 背景

DeepSeek 模型名会随版本升级而变化，直接把模型名或 SDK 写进领域代码会让升级牵涉
业务逻辑，也违反领域层不依赖供应商 SDK 的边界。

## 决策

通过 ModelGateway 端口抽象模型调用；DeepSeek 适配器使用其 OpenAI 兼容协议，Base URL、
API Key、模型名、超时、并发与预算均经环境变量配置，具体模型名不进领域代码
（SPEC.md 第 9.1、9.2 节）。

## 后果

- 收益：切换模型或供应商不需改领域代码；调用可记录、限流与预算管控；
- 代价：需要自建 JSON 解析、Schema 校验与错误分类（阶段 F）；
- 不允许静默切换到其他供应商或模型（SPEC.md 第 11.2 节）。

## 重评条件

当需要多供应商路由或本地模型推理，且 ModelGateway 抽象能继续承载时扩展；否则评估
新的抽象，但领域层仍不得直接依赖供应商 SDK。
