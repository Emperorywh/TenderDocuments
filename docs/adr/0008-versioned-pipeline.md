# ADR-008 解析器、模型、提示词、Schema 和规则版本化

- 状态：Accepted
- 日期：2026-07-22

## 背景

解析器、OCR、模型、提示词、Schema 与规则都会随时间升级。若不版本化，升级会在后台
静默改写历史结果，破坏报告不可变性与可审计性（SPEC.md 第 11.5 节）。

## 决策

每类产物都携带版本：解析器、OCR、模型调用（模型名、提示词 ID 与版本、Schema ID
与版本、规则包版本）。正式结论记录所用的全部版本，升级不回写历史；用户显式重分析后
才生成新结果（SPEC.md 第 9.1、9.3、11.5 节）。

## 后果

- 收益：历史结果稳定，升级可控、可回滚、可解释；
- 代价：结果与版本绑定，查询与迁移需考虑版本兼容；
- 模型名不得写入领域代码，经配置注入（ADR-012）。

## 重评条件

当版本矩阵过于复杂导致维护成本过高，重新评估版本粒度，但保留“升级不改写历史”原则。
