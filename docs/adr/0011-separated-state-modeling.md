# ADR-011 文件接入、分析运行、任务和完整性分离建模

- 状态：Accepted
- 日期：2026-07-22

## 背景

文件接入状态、分析运行状态、任务执行状态与完整性是不同维度的概念。用一个字段混装
（如“处理中”既表示上传又表示分析）会导致状态机不可推理、迁移与恢复逻辑混乱。

## 决策

文件接入（DocumentVersion）、分析运行（AnalysisRun）、任务（AnalysisTask）与完整性
（COMPLETE/INCOMPLETE）各自独立建模与表达，状态转换集中定义，不混用（SPEC.md 第
5.2、5.3 节）。业务状态机不依赖操作记录反推（SPEC.md 第 6.2 节）。

## 后果

- 收益：每个状态机可独立测试、推理与恢复；
- 代价：模型字段更多，需要在应用层维护各状态机的一致性；
- 进度展示由数据库计算，不展示假进度。

## 重评条件

当证明某两个状态维度实际始终联动、分离反而增加复杂度且无独立价值时，可重新评估，
但不得退回“单字段混装”。
