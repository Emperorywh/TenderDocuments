# ADR-014 首版采用无应用鉴权的单实例单工作区模式

- 状态：Accepted
- 日期：2026-07-22

## 背景

首版范围明确排除用户、组织、成员、角色、会话、Token、RBAC 与多租户（SPEC.md 第
3.3、4 节）。在应用内实现半套鉴权会增加复杂度且制造假安全感。

## 决策

首版为单实例、单工作区、无应用鉴权：不建立身份/组织/角色/会话/Token 数据模型，业务
表不含 organization_id/user_id/created_by/reviewed_by，不启用 RLS。访问边界由受控
内网、VPN 或等价网络措施承担（SPEC.md 第 4.1、4.2 节）。

## 后果

- 收益：领域与数据模型显著简化，无身份残留；
- 代价：生产入口必须置于可信网络边界，对象默认私有、端口不公开；
- 未来引入身份/多租户须作为新版本重新设计数据迁移，不预埋兼容分支。

## 重评条件

当系统需要服务多个相互隔离的客户/组织，或必须在应用层区分操作者权限时，作为新版本
重新设计身份边界与数据迁移；在此之前不为未来需求预埋分支。
