# ADR-003 API 与 Worker 同库不同进程

- 状态：Accepted
- 日期：2026-07-22

## 背景

解析、OCR、模型抽取与报告渲染为长耗时任务，不能占用 API 请求线程；但它们与 API
共享同一领域模型、仓储与端口定义，重复维护两套代码会迅速漂移。

## 决策

API 与 Worker 共享同一代码库与领域/应用层，作为不同进程运行：API 进程处理请求，
Worker 进程消费 Celery 队列执行任务。OCR 等重负载使用独立 Worker 镜像/进程
（PLAN.md 第 13.1 节）。

## 后果

- 收益：领域规则与仓储单一来源，任务与请求行为一致；
- 代价：API 镜像需避免包含不必要的 OCR/GPU 运行时，需分层构建镜像；
- Celery 仅做调度，不持有业务状态（见 ADR-005）。

## 重评条件

当 API 与 Worker 的依赖差异（如 OCR 运行时体积）导致共享代码库成本高于收益，或
部署节奏严重冲突时，重新评估是否将 Worker 拆为独立代码库。
