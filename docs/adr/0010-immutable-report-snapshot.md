# ADR-010 报告采用不可变快照

- 状态：Accepted
- 日期：2026-07-22

## 背景

发布的报告是交付物，下游（PDF、复核记录、用户决策）依赖其稳定。若报告随实时数据
变化，已下载的 PDF 与系统状态会不一致，审计无法对齐。

## 决策

发布时生成不可变 ReportSnapshot，记录输入指纹、Schema 版本与全部结论；PDF 作为后续
幂等任务基于快照渲染。新文件进入只把旧快照标记 OUTDATED，不改其内容（SPEC.md 第
6.16、6.17、11.5 节）。

## 后果

- 收益：交付物稳定、可审计、可重生成；
- 代价：快照存储与 Schema 校验增加成本；
- 相同输入重复发布返回同一快照（幂等）。

## 重评条件

当快照体积或 Schema 变更频率导致存储与迁移成本过高，重新评估快照存储策略，但保留
“发布即不可变、升级不改历史”原则。
