# ADR-004 PostgreSQL 是业务状态事实来源

- 状态：Accepted
- 日期：2026-07-22

## 背景

系统涉及多状态机（文件接入、分析运行、任务、复核、报告），需要严格的事务、唯一
约束与可恢复性。若把权威状态放在 Redis 或 Celery Result Backend，进程重启或消息
丢失都会造成不可恢复的业务漂移。

## 决策

PostgreSQL 是业务状态唯一事实来源；所有状态转换、唯一性约束与完整性都在数据库中
强制（SPEC.md 第 7、12.2 节）。Redis 仅作队列与缓存，Celery Result Backend 不参与
完整性判断，重启后从 PostgreSQL 恢复。

## 后果

- 收益：状态可恢复、可审计，约束有数据库兜底；
- 代价：写路径必须经 PostgreSQL，需关注长事务与连接池；
- Redis 丢失数据不影响业务正确性。

## 重评条件

当单一 PostgreSQL 实例无法承载吞吐或可用性要求，且已穷尽读写分离/分区等手段时，
重新评估是否引入读写分离或分库（仍以关系库为事实来源）。
