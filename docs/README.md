# docs

项目文档目录。

## 职责

承载 ADR（架构决策记录）、部署文档、模块说明、运行手册与验收记录等开发与运维文档。

`SPEC.md`、`PLAN.md`、`TASKS.md` 和 `PROGRESS.md` 作为项目顶层规划与追踪文档，保留在仓库根目录，不移动到本目录。

## 边界

- 每份 ADR 至少包含背景、决策、后果和重评条件（`PLAN.md` 第 4 节，对应 `TASKS.md` 的 `A-020`）；
- 文档不得包含密钥、Token、签名 URL 或完整招标原文（`SPEC.md` 第 6.2、12.3 节）。

具体内容由 `TASKS.md` 的 `A-020`（ADR）、`A-023`（模块 README 模板）与阶段 K（部署文档）等任务按依赖逐步补充。
